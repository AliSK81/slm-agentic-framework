WHY 0.2.x PATCHES FAIL (mechanism summary)
================================================================================

A. One orchestration for everything: salam and create foo.py both run planner+executor LangGraph up to 15 steps.

B. Dual verification: framework SessionOutcome.solved vs analyze_turn_effects.satisfied — often disagree;
   patches force unresolvable, fallbacks, or auto-undo after the fact.

C. REPL "detail" is scraped from decision log (terminate rationale, effect_sink tool text) via _best_answer()
   with heuristics — not a first-class contract. Empty detail + ok is possible.

D. classify_goal() regex drives hints, read-only mode, step budget, verification, fallbacks — unbounded whack-a-mole.

E. Deterministic fallbacks (read_file, README summary) hide SLM failures and produce wrong answers for wrong goal types.

F. Test pyramid inverted: 67 mocked tests green; L3 live is optional minimal smoke; user IS the integration test.
